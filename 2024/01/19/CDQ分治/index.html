CDQ分治

CDQ分治

参考资料：

CDQ 分治 - OI Wiki (oi-wiki.com)

\(\textrm{CDQ}\) 分治主要用于解决三类问题： 1. 解决和点对有关的问题 2. \(\textrm{1D}\) 动态规划的优化与转移 3. 通过 \(\textrm{CDQ}\) 分治， 将一些动态问题转化为静态问题

解决和点对有关的问题

这类问题多数类似于\(\lceil\)给定一个长度为 \(n\) 的序列，统计一些有特性的点对 \((i,j)\) 的数量或者找到一对点 \((i,j)\) 使得一些函数的值最大\(\rfloor\)。

算法流程如下：

  1. 找到这个序列的终点 \(mid\)

  2. 将所有点对 \((i, j)\) 划分为 \(3\) 类：

    1. \(1\leq i\leq mid,1\leq j\leq mid\) 的点对；

    2. \(1\leq i\leq mid,mid+1\leq j\leq n\) 的点对

    3. \(mid+1\leq i\leq n,mid+1\leq j\leq n\) 的点对

  3. 将 \((1,n)\) 这个序列拆成两个序列 \((1,mid)\) 和 \((mid+1,n)\)。此时第一类点对和第三类点对都在这两个序列之中。

  4. 递归地处理这两类点对

  5. 设法处理第二类点对

可以发现 \(\textrm{CDQ}\) 分治的思想就是不断地把点对通过递归的方式分给左右两个区间。 在实际应用时，通常使用一个函数 solve(l, r) 处理 \(l\leq i\leq r,l\leq j\leq r\) 的点对。上述算法流程中的递归部分便是通过 solve(l, mid) 与 solve(mid + 1, r) 来实现的。剩下的第二类点对则需要额外设计算法解决。

例题：
三维偏序：

给定一个序列，每个点有 \(a_i,b_i,c_i\) 三个属性，试求这个序列里有多少对点对 \((i,j)\) 满足 \(a_j\leq a_i\) 且 \(b_j\leq b_i\) 且 \(c_j\leq c_i\) 且 \(j\neq i\)。

思路：

三维偏序是 \(\textrm{CDQ}\) 分治的经典问题，题目中要求统计序列里点对的个数，那就尝试使用 \(\textrm{CDQ}\) 分治。

首先将序列按照 \(a\) 排序。

假设我们已经写好了 solve(l,r)，并通过递归搞定了 solve(l, mid) 和 solve(mid + 1, r)。现在我们要做的就是统计满足 \(l\leq i\leq mid,mid+1\leq j\leq r\) 的点对 \((i,j)\) 种，有多少个点对还满足题目的限制条件。

思考可以发现，由于我们前面已经对 \(a\) 进行排序了，那么 \(a_i<a_j\) 就能转化成 \(i<j\)，既然 \(i\) 比 \(mid\) 小， \(j\) 比 \(mid\) 大，那 \(i\) 肯定要比 \(j\) 要小。现在还剩下两个限制条件：\(b_i<b_j\) 与 \(c_i<c_j\)，根据这个限制条件我们就可以枚举 \(j\) ，求出有多少个满足条件的 \(i\)。

为了方便枚举，我们把 \((l,mid)\) 和 \((mid+1,r)\) 中的点全部按照 \(b\) 的值从很小到大排序，之后依次枚举每一个 \(j\) ，把所有 \(b_i<b_j\) 的点 \(i\) 全部插入到某种数据结构里（这里可以选用树状数组），此时只要查询有多少个点的 \(c\) 的值是小于 \(c_j\) 的，我们就求出了对于这个点 \(j\) ，有多少个 \(i\) 可以合法匹配它了。

当我们插入一个 \(c\) 值等于 \(x\) 的点时，我们就令树状数组的 \(x\) 这个位置单点 +1，而查询树状数组有多少个点小于 \(x\) 的操作实际上就是在求前缀和，只要实现对所有的 \(c\) 做离散化，复杂度就是正确的。

对于每个 \(j\)，我们都需要将所有的 \(b_i<b_j\) 的点 \(i\) 插入树状数组中，由于所有的 \(i\) 和 \(j\) 都已事先按照 \(b\) 值排好序，这样只要以双指针的方式在树状数组里插入点，那么插入操作就能从 \(O(n^2)\) 降到 \(O(n)\)。

那么通过这样的算法流程，我们就用 \(O(nlogn)\) 的时间处理完了第二类点对的信息了，此时算法的时间复杂度就是 \(T(n)=T(\lfloor\frac{n}{2}\rfloor)+T(\lceil\frac{n}{2}\rceil)+O(nlogn)=O(nlog^2n)\)。

动态逆序对：

对于序列 \(a\)，它的逆序对数定义为集合 \(\{(i, j)|i<j \wedge a_i>a_j\}\) 中元素个数。 现给出 \(1\sim n\) 的一个排列，按照某种顺序依次删除 \(m\) 个元素，你的任务是在每次删除一个元素之前统计整个序列的逆序对数。

思路：给每一个元素加入在加入一维删除时间 \(time\)，那么对答案做出贡献的必然是 \(time_i < time_j, i<j,a_i>a_j\) 和 \(time_i<time_j,i>j,a_i<a_j\) 两部分，明显看出就是三维偏序的板子，分别做 \(CDQ\) 分治即可。

CDQ分治优化 1D/1D 动态规划的转移

1D/1D 动态规划指的是一类特定的 DP 问题，该类题目的特征是 DP 数组是一位的，转移是 \(O(n)\) 的。如果条件良好的话，又是啥可以通过 CDQ 分治来把他们的时间复杂度由 \(O(n^2)\) 降至 \(O(nlog^2n)\)。

例如，给定一个序列，每个元素由两个属性 \(a,b\)。我们希望计算一个 DP 式子的值，它的转移方程如下： \[ dp_i = 1+max_{j=1}^{i-1}dp_j[a_j<a_i][b_j<b_i] \] 这是一个二维最长上升子序列的 DP 方程，即只有 \(j<i,a_j<a_i,b_j<b_i\) 的点 \(j\) 可以更新点 \(i\) 的 DP 值。 直接转移显然就是 \(O(n^2)\) 的。以下是使用 CDQ 分治优化转移过程的讲解。

首先发现 \(dp_j\) 转移到 \(dp_i\) 这种转移关系也是一种点对间的关系，因此也可以使用类似 CDQ 分治处理点对关系的方式来进行处理。

考虑上面给出的思路，假设现在处理的区间是 \((l,r)\)，那么算法的流程大概就是：

  1. 如果 \(l=r\) ，说明 \(dp_r\) 值已经被计算好了，直接令 \(dp_r++\) 然后返回即可。
  2. 递归使用 \(solve(l,mid)\)；
  3. 处理所有 \(l\leq j\leq mid,mid+1\leq i\leq r\) 的转移关系；
  4. 递归使用 \(solve(mid+1, r)\)

观察发现第三步的做法和求三维偏序的步骤差不多。首先 \(j<i\) 的限制条件已经不用继续考虑了，这是显然的事情，所以我们只需要将所有的点 \(i\) 和点 \(j\) 按 \(a\) 值进行排序，然后用双指针的方式将 \(j\) 点插入到树状数组里，最后查一下前缀最大值更新一下 \(dp_i\) 就可以。
